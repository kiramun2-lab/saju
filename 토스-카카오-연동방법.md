# 토스페이먼츠 + 카카오 로그인 연동 방법

## 0. 환경 변수 파일 만들기

프로젝트 루트에서:

```bash
cp .env.example .env.local
```

`.env.local`을 열어서 아래 값들을 **본인 키**로 채우면 됩니다. (이 파일은 Git에 안 올라갑음)

---

## 1. 토스페이먼츠 (결제)

### 1) 키 발급

1. [토스페이먼츠 개발자센터](https://developers.tosspayments.com) 로그인
2. **내 개발정보** → **API 키**
3. 복사해서 쓸 것:
   - **클라이언트 키** (test_ck_... 또는 live_ck_...) → 결제창 띄울 때 사용
   - **시크릿 키** (test_sk_... 또는 live_sk_...) → 서버에서 결제 승인할 때만 사용

### 2) .env.local에 넣기

```env
NEXT_PUBLIC_TOSS_CLIENT_KEY=test_ck_여기에_클라이언트키
TOSS_SECRET_KEY=test_sk_여기에_시크릿키
```

- `NEXT_PUBLIC_` 붙은 건 브라우저에 노출돼도 됨 (결제 위젯용)
- `TOSS_SECRET_KEY`는 **절대** 프론트/공개에 두지 말고, 서버(API 라우트)에서만 사용

### 3) 동작 흐름

1. 사용자가 **결제 페이지** (`/payment`)에서 990원 결제 클릭
2. 토스 결제창 뜸 → 결제 완료 시 **성공 URL**로 리다이렉트  
   - 성공: `/payment/success?paymentKey=...&orderId=...&amount=...`
3. 성공 페이지에서 **서버 API** `/api/payments/confirm` 호출  
   - 이 API가 토스 **결제 승인** 호출 (시크릿 키 사용)
4. 실패 시 `/payment/fail`로 이동

테스트 시에는 **test_ck / test_sk** 쓰고, 실제 오픈 시 **live_** 키로 바꾸면 됨.

---

## 2. 카카오 로그인

### 1) 앱·키 확인

1. [카카오 개발자 콘솔](https://developers.kakao.com) 로그인
2. **앱** 선택 (없으면 만들기)
3. **앱 키** 메뉴에서:
   - **JavaScript 키** → 로그인 버튼/스크립트용
   - **REST API 키** → 서버에서 인가 코드 ↔ 토큰 교환용

### 2) Redirect URI 등록 (필수)

1. 같은 앱에서 **카카오 로그인** (또는 **제품 설정**)
2. **Redirect URI** 등록:
   - 로컬: `http://localhost:3000/oauth/callback/kakao`
   - 배포 도메인: `https://read-destiny.com/oauth/callback/kakao` (또는 본인 도메인)
3. **저장** 꼭 하기

### 3) .env.local에 넣기

```env
NEXT_PUBLIC_KAKAO_JS_KEY=여기에_JavaScript_키
KAKAO_REST_API_KEY=여기에_REST_API_키
NEXT_PUBLIC_KAKAO_REDIRECT_URI=http://localhost:3000/oauth/callback/kakao
```

**중요:** `NEXT_PUBLIC_KAKAO_REDIRECT_URI`는 카카오 콘솔에 등록한 Redirect URI와 **완전히 동일**해야 함 (끝에 `/` 유무까지 맞추기).

### 4) 동작 흐름

1. **로그인 페이지** (`/login`)에서 "카카오로 3초만에 시작하기" 클릭
2. 카카오 로그인/동의 화면 → 동의 후 **Redirect URI**로 돌아옴  
   - 예: `/oauth/callback/kakao?code=...`
3. 이 프로젝트는 `code`를 `/api/auth/kakao/token`에 보내서 토큰 받고, 로그인 처리 후 `/mypage`로 이동

---

## 3. 배포(운영) 시

### Vercel 환경 변수

1. Vercel 프로젝트 → **Settings** → **Environment Variables**
2. 아래 변수 추가 (Production / Preview 필요한 대로 선택):

| 이름 | 값 | 비고 |
|------|-----|------|
| `NEXT_PUBLIC_TOSS_CLIENT_KEY` | (토스 클라이언트 키) | 운영이면 live_ck_... |
| `TOSS_SECRET_KEY` | (토스 시크릿 키) | 운영이면 live_sk_... |
| `NEXT_PUBLIC_KAKAO_JS_KEY` | (카카오 JavaScript 키) | |
| `KAKAO_REST_API_KEY` | (카카오 REST API 키) | |
| `NEXT_PUBLIC_KAKAO_REDIRECT_URI` | `https://read-destiny.com/oauth/callback/kakao` | 실제 도메인으로 |

3. **저장** 후 **Redeploy** 한 번 하면 적용됨.

### 배포 후 체크

- **토스**: 운영 키면 실제 결제 테스트 한 번 하기
- **카카오**: 사용하는 **운영 도메인**을 콘솔 Redirect URI에 추가했는지 확인

---

## 4. 한 줄 요약

- **토스**: 개발자센터에서 클라이언트 키·시크릿 키 받아서 `.env.local`에 넣기 → 결제 페이지에서 바로 테스트 가능  
- **카카오**: 콘솔에서 JavaScript 키·REST API 키 받고, **Redirect URI 먼저 등록**한 뒤 `.env.local`에 같은 URI 포함해서 넣기 → 로그인 버튼 동작 확인

둘 다 로컬에서는 `.env.local`, 배포에서는 Vercel Environment Variables에 같은 이름으로 넣으면 됨.
